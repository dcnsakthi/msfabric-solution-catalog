## Getting Started

1. **Open an issue first.** Before writing code, create an issue (or use the `New catalog` template for catalogs) so maintainers can confirm the change fits the project's direction.
2. **Discuss larger changes** with maintainers via an issue or draft PR to align on scope before investing effort.
3. **Fork the repo** and create a feature branch from `main`.
4. **Set up your environment** by following the contributing guide for the project you're working on (linked above).

## Commit and PR Conventions

- PRs must follow [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) in the headline (e.g., `feat: add spark-monitoring catalog`).
- Use the [PR template](.github/pull_request_template.md) — include a brief rationale and note any user-facing impacts.
- Keep commits focused; avoid formatting-only churn.

## Standards

Please read [STANDARDS.md](src/msfabric_solution_catalog/STANDARDS.md) for catalog design and quality expectations before opening a pull request.
